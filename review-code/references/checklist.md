# 代码审查检查清单（按关注点分类）

## 安全
- [ ] 所有用户输入都经过校验和清理
- [ ] SQL/NoSQL 查询使用参数化语句
- [ ] 受保护操作都有认证检查
- [ ] 授权检查验证资源所有权
- [ ] 代码中没有密钥、令牌或密码
- [ ] 日志和错误信息中无敏感数据
- [ ] 文件上传有类型/大小校验
- [ ] 公共端点已应用限流

## 性能
- [ ] 无 N+1 查询模式
- [ ] 数据库查询使用了合适的索引
- [ ] 耗时操作为异步或已缓存
- [ ] 无内存泄漏（事件监听器、闭包）
- [ ] 大数据集已分页
- [ ] 避免将大文件加载到内存

## 可靠性
- [ ] 外部服务调用有超时设置
- [ ] 对瞬态失败有退避重试逻辑
- [ ] 对级联失败有熔断机制
- [ ] 依赖故障时优雅降级
- [ ] 多步操作使用数据库事务

## 可维护性
- [ ] 命名表达意图（避免 `data`、`temp`、`process`）
- [ ] 函数只做一件事
- [ ] 魔法数字已命名为常量
- [ ] 复杂条件已提取/解释
- [ ] 注释解释为什么，而非做什么
- [ ] 无被注释掉的代码

## 测试
- [ ] 主流程已覆盖
- [ ] 需求中的边界场景已覆盖
- [ ] 错误路径已覆盖
- [ ] 测试相互独立（无共享可变状态）
- [ ] Mock 在合适的边界
- [ ] 测试名称描述行为，而非实现
