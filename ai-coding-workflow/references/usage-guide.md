# AI 编码工作流使用指南

## 快速开始

1. 在对话中粘贴你的 PRD 或功能描述
2. 说："针对这个 PRD 启动完整开发流程"
3. Kimi 会引导你逐个阶段推进，每个阶段结束后请求你的确认

## 单阶段独立使用

你也可以直接触发某个具体阶段：

| 阶段 | 触发语句 |
|------|---------|
| 需求分析 | "分析这些需求" / "结构化这个 PRD" |
| 技术方案设计 | "设计这个功能的技术方案" |
| 任务拆解 | "把这个设计拆成任务" / "任务分解" |
| 编码 | "实现任务 T-03" / "编写 OrderService" |
| 代码审查 | "审查这段代码" / "对订单功能做代码审查" |
| 知识归档 | "给这个功能写文档" / "归档设计决策" |

## 示例对话流程

**用户**： [粘贴 PRD]

**用户**： 针对这个 PRD 启动完整开发流程

**Kimi**： ## 阶段 1 完成：需求分析
...（结构化需求）...

请审阅。在继续技术方案设计前，有需要澄清的地方吗？

**用户**： 没问题，继续

**Kimi**： ## 阶段 2 完成：技术方案设计
...（技术方案）...

请审阅方案。在拆解任务前，有任何修改意见吗？

**用户**： 批准

**Kimi**： ## 阶段 3 完成：任务拆解
...（任务列表）...

准备开始编码？建议从第 1 层（基础层）开始。

**用户**： 开始编码

**Kimi**： ## 阶段 4：编码 - 任务 T-01
...（实现迁移和类型定义）...

任务 T-01 完成。继续 T-02？

...（继续完成所有任务）...

**Kimi**： ## 阶段 5 完成：代码审查
...（审查报告）...

所有严重问题已修复。继续知识归档？

**用户**： 是

**Kimi**： ## 阶段 6 完成：知识归档
...（文档）...

工作流结束。文档已保存到 `docs/knowledge-base/`。

## 使用技巧

- **迭代优化**：如果你发现方案需要修改，随时可以说"回到阶段 2"
- **部分流程**：紧急修复时，直接说"跳过设计，直接实现 X"
- **并行编码**：如果任务间没有依赖，可以说"并行实现 T-02 和 T-03"
- **审查聚焦**：可以说"只审查安全性"或"只审查性能"来缩小审查范围
